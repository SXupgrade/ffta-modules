# Internationalization (i18n)

## Principle

- Default language: English.
- All user-visible text goes through `app.t(key)`.
- French labels live only in `fr.json` — never hardcoded in JS or PHP components.

## File structure

```
modules/my-module/i18n/
  en.json   — English (required)
  fr.json   — French (required for FFTA deployment)

core/i18n/
  en.json   — Core platform strings
  fr.json   — Core platform strings in French
```

## JSON format

```json
{
  "title": "My Module",
  "navigation": { "title": "My Module" },
  "actions": { "save": "Save", "cancel": "Cancel" },
  "errors": { "loadFailed": "Failed to load data." }
}
```

The file content is registered as the namespace body. The key path after the namespace:

- `app.t('my-module.title')` → namespace `my-module`, path `title`
- `app.t('my-module.navigation.title')` → namespace `my-module`, path `navigation.title`

## Registering a namespace

In `module.mount.js`:

```js
import en from './i18n/en.json' assert { type: 'json' };
import fr from './i18n/fr.json' assert { type: 'json' };

app.i18n.registerNamespace('my-module', { en, fr });
```

## Using translations

```js
app.t('my-module.title')
app.t('my-module.navigation.title')
app.t('my-module.errors.loadFailed')
```

With interpolation (uses `{placeholder}` syntax):

```js
app.t('my-module.warnings.tooMany', { max: 8 })
// fr.json: "Un maximum de {max} éléments."
```

## Language detection

The Ianseo language adapter reads the language selected by Ianseo through `SelectLanguage()` injected by `index.php` into `window.__FFTA_IANSEO_LANGUAGE__`. It then falls back to the module shell data attribute, the Ianseo `UseLanguage` cookie, and finally `navigator.language`. Languages are normalized to the available translation namespace, for example `FR`/`fr-FR` becomes `fr`.

## Fallback

If a key is not found in the current language, the service falls back to `en`. If it is not found in `en` either, the raw key string is returned (useful for spotting missing translations during development).
